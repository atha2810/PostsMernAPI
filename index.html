Welcome to DevSnippetsAPI

Made using NodeJS, ExpressJS, MongoDB.

This API provides a collection of informative and educational posts authored by Atharva lakhamade, covering various topics in software development.

Getting Started

  1. Use ----> /posts <---- after your browser Url to Get all posts From Api.
  2. Use ----> /posts:specificPostID <---- after your browser Url to Get a single specific post From Api.
  3. Can also Create post or Delete post if you haved Registered or Login

Registration and Login

API can be used for User Registration and Login with Postman or any other API testing apps.

Register a User
Send a POST request to /api/user/register with the following JSON body:
{ "name": "Your Name", "email": "your@email.com", "password": "yourpassword" }
Login
Send a POST request to /api/user/login with the following JSON body:
{ "email": "your@email.com", "password": "yourpassword" }

Access Protected Routes

You should receive a response with a JWT token in the auth-token header.

For routes that require authentication:

  • Add the JWT token received from the login response to the auth-token header of your requests.

Use of Tokens with Postman or API Testing App:

  1. Obtain a token through your API's authentication process (e.g., login or registration).
  2. Copy the token value and set it in the headers of your API request. In Postman, add a header with key auth-token and the token value.
  3. Send your request to a protected route that uses the provided middleware function. The middleware will verify the token and allow access if it's valid. If the token is invalid or missing, it will return an error message.

Steps:

  1. Obtain a token through your authentication process.
  2. Copy the token value.
  3. Open Postman or your API testing app.
  4. Create a new request and select the appropriate HTTP method.
  5. Set the request URL to a protected route that uses the middleware.
  6. Add a new header with key auth-token and the token value as the header value.
  7. Send the request.

CRUD Operations on Posts

Use the following endpoints for CRUD operations on posts:

  • GET /posts: Get all posts.
  • POST /posts: Create a new post.
  • GET /posts/:postId: Get a specific post by ID.
  • PATCH /posts/:postId: Update a specific post by ID.
  • DELETE /posts/:postId: Delete a specific post by ID.
